# Отчёт о конкурсном решении (минимальный шаблон)

Работа участника оценивается по файлу `src_solution/docs/solution.md`.

Используйте этот шаблон по принципу «копируй и заполняй». Разделы ниже покрывают обязательный минимум для критериев.

## 1. Архитектура и границы ДВБ

- Где находится доверенный код (`src_solution/abu/tcb`) и недоверенный (`src_solution/abu/other`).
- Как реализованы process-изоляция и контроль IPC.
- Какие домены являются критичными для SG.

## 2. Политики и цели безопасности (SG/SA)

- Список SG/SA.
- Как политики в `security_monitor` поддерживают SG.
- Какие операции разрешены, какие запрещены (`default deny`).

## 3. Тесты и результаты

- Команды запуска: `make tests-all`, `pipenv run pytest src_solution/tests -q`.
- Краткий итог прохождения тестов.
- Сквозной сценарий ЦР–АБУ (e2e) и его статус.

### 3.1 Таблица тестов безопасности (обязательно для C13)

| Цель (SG) | Файлы тестов в `src_solution/tests` | Проверяемые файлы в `src_solution` | Комментарий |
|-----------|--------------------------------------|-------------------------------------|-------------|
| SG_...    | `src_solution/tests/...`             | `src_solution/...`                  | ... |

## 4. Сертификация

- Команды: `make prepare-cert-bundle-solution`, `make certify-abu-solution`.
- Результат: успешно/неуспешно.
- Стоимость и идентификатор сертификата.

## 5. Итог оценки

- Результат `make evaluate-score` (по возможности с ключевыми Cxx).
- Краткий список сильных и слабых мест решения.

## 6. Диаграммы (опционально, но желательно)

- Диаграмма архитектуры политики.
- Диаграмма последовательности основного сценария.
