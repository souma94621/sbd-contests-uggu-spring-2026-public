# Учебный пример «Светофор» (`references/traffic_light_demo`)

## Когда читать

- **Обязательно:** нет.
- **Когда:** если нужна наглядная опора по изоляции доменов, монитору безопасности и IPC-маршрутам.
- **Можно пропустить:** да, если архитектура уже спроектирована и подтверждена тестами.

## Что это за пример

В каталоге `references/traffic_light_demo` размещён локальный учебный пример в формате Jupyter Notebook:

- `references/traffic_light_demo/cyberimmunity-traffic-lights-example.ipynb`
- изображения к диаграммам в `references/traffic_light_demo/images/`

Пример демонстрирует:

- разделение ролей компонентов по зонам ответственности;
- применение монитора безопасности как единой точки контроля взаимодействий;
- проверку маршрутов сообщений через явно заданные правила.

## Как работать с примером

1. Сначала прочитайте markdown-блоки с описанием архитектурной политики.
2. Затем просмотрите кодовые блоки, где показаны маршруты сообщений и проверки.
3. Сопоставьте роли из примера с доменами вашего решения в `src_solution/abu/tcb` и `src_solution/abu/other`.
4. Перенесите только принципы (границы, политики, проверки), а не буквальную структуру классов.

## Когда пример можно не использовать

- Если у вас уже есть обоснованная модель доменов по TARA.
- Если монитор безопасности и IPC-политики реализованы и покрыты security-тестами.
- Если `make evaluate-score` стабильно подтверждает уровни по `C18`, `C19`, `C24`, `C25`.
