# Сквозной сценарий эксплуатации АБУ (версия прототипа v1)

Сценарий описывает минимальную цепочку, которую должна отработать первая версия прототипа ЦР и АБУ.

## Шаги

1. **Регистрация АБУ в ЦР**  
   Оператор или интеграционный клиент регистрирует установку: идентификатор `rig_id`, базовый URL HTTP API АБУ, при наличии — идентификатор сертификата (`certificate_id`), полученный после сертификации у Регулятора.

2. **Проверка допуска**  
   ЦР проверяет политику `CR_CERT_POLICY`:
   - в режиме **strict** миссия разрешена только при действительном сертификате (запрос к Регулятору);
   - в режиме **permissive** миссия может быть выдана без сертификата (с предупреждением в ответе API).

3. **Выдача задания**  
   ЦР создаёт миссию (целевая глубина, ограничения) и передаёт её в АБУ методом `POST` на API АБУ.

4. **Исполнение**  
   АБУ принимает миссию, переводит внутреннее состояние в режим работы, периодически отдаёт статус и телеметрию. При необходимости вызываются модули псевдо-ИИ (подсказки режима, оценка аномалий) — в v1 они не изолированы от остального кода.

5. **Завершение**  
   Миссия завершается успешно, по аварии или отмене; ЦР получает итоговый статус через опрос или последний снимок телеметрии.
