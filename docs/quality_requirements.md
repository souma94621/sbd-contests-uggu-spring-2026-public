# Требования к качеству проекта

## Общие 

- **Баллы по критериям** выставляет скрипт `make evaluate-score` ([`scripts/evaluate_contest_score.py`](../scripts/evaluate_contest_score.py)) и при необходимости жюри. **Имена тестовых функций pytest и комментарии в коде не являются строками таблицы баллов** — не ориентируйтесь на них как на официальные значения C01–C22.
- Python код содержит docstring для каждого модуля и функций
- Код соответствует практикам и рекомендациям SOLID, KISS, PEP8, clean code
- Работают юнит, модульные и интеграционные тесты
- тесты запускаются командами из Makefile, команда make tests-all запускает все тесты
- для тестов используется pytest
- все изменения фиксируются в changelog

## Артефакты

- документация в каталоге docs
- исходный код первоначальной версии решения (АБУ) в каталоге src_starting_point
- исходный код тестов находится в подкаталогах tests/*

## Смежные системы

- прототип надсистемы "Цифрового двойника" находится в каталоге external_systems/digital_mine
- прототип "Регулятора" находится в каталоге external_systems/regulator

## Окружение

- Все **Python-пакеты** (включая зависимости для тестов, линтеров и сервисов) устанавливаются **только** в изолированном окружении **Pipenv** (`pipenv install --dev`; виртуальное окружение в `.venv/` при `PIPENV_VENV_IN_PROJECT=1`, как в Makefile). Альтернатива — **Poetry** с локальным venv или **`python -m venv .venv`** с установкой из зафиксированного списка.
- Источник зависимостей — **[Pipfile](../Pipfile)**; для воспроизводимых сборок — **[Pipfile.lock](../Pipfile.lock)** (его следует коммитить в репозиторий).
- **Запрещено** ставить пакеты в системный интерпретатор: не использовать `pip install` / `pip3 install` без активированного venv/Pipenv и не использовать флаги вроде `--break-system-packages` для обхода ограничений дистрибутива.
- В Makefile и в документации команды приводятся через **`pipenv run`** (например `pipenv run pytest`) либо `./scripts/bootstrap_pipenv.sh` для первичной установки зависимостей.

## Требование к версионированию

- файлы сохраняются в git репозитории в комментариями о сделанных изменениях
- комментарии к изменениям в git оформляются согласно conventional commits specification 
- комментарии к коду сделаны на русском языке, отсутствуют нечитаемые символы
- в репозитории храни только исходный код, необходимые ресурсы, скрипты и документацию, не храни временные и сгенерированные в результате выполнения системы файлы

## Не цели

- не меняй содержимое каталога `src_solution` в ходе сопровождения учебного прототипа без отдельной договорённости; конкурсное решение участники размещают в `src_solution/` согласно [contest_regulations.md](contest_regulations.md).
