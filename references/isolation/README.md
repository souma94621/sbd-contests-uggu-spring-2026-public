# Пример `isolation`

Минимальный пример process-изоляции доменов и асинхронного IPC через очередь.

## Что показывает пример

- разнесение доменов по процессам;
- обмен событиями через `Event`;
- отсутствие прямых вызовов между доменами.

## Быстрый запуск

```bash
pipenv run pytest -q references/isolation/tests
```

## Шаг -> файл -> тест -> критерий

| Шаг | Файл | Тест | Критерий (ориентир) |
|---|---|---|---|
| Process-изоляция | `ipc.py` | `tests/test_isolation_queues.py::test_domain_process_runs_and_stops` | C19 |
| Асинхронный IPC | `ipc.py` | `tests/test_isolation_queues.py::test_cross_process_roundtrip_async` | C19 |
| Базовая обработка события | `tcb_controller/domain.py`, `sensor_adapter/domain.py` | `tests/test_isolation_queues.py::test_unknown_operation_returns_error` | C18 (базовый уровень) |

## Обязательные негативные проверки

- неизвестная операция возвращает ошибку;
- событие не обрабатывается прямым вызовом функции другого домена;
- процесс домена корректно завершается и не оставляет «висящих» задач.

## Анти-паттерны (накрутки)

- делать разделение только каталогами без процессов;
- подменять асинхронный обмен прямым вызовом между доменами;
- считать этот пример готовой реализацией `security_monitor` (здесь только изоляция, без полного policy-слоя).

Дальше используйте `references/secure_ipc` для примера с мандатным контролем и маршрутными политиками.

## Чек-лист перед переносом в `src_solution`

- тесты примера проходят;
- домены действительно запускаются в отдельных процессах;
- в итоговом решении тесты размещены только в `src_solution/tests/**`;
- итоговый self-check решения: `make tests-all`, `make evaluate-score`, `make certify-abu-solution`.
